Closures

What is it?

A closure is when a nested function, a function inside of a function, as access to its outer functions variables and paramaters, as well as the variables of the global scope. To understand closures, you first must know about scope.

Why use it?

  • You can associate some data with a function that operates on that data.
  • They allow you to emulate private methods which is useful for restricting access to code as well as managing thr global namespace.

How to use it? 
        
        function makeAdder(x) {
          return function(y) {
            return x + y;
          };
        }

        var add5 = makeAdder(5);
        var add10 = makeAdder(10);

        console.log(add5(2));  // 7
        console.log(add10(2)); // 12

In the above example, function makeAdder() takes an argument x and returns a function, which takes an argument y and adds both of the arguments together.

add5 and add10 are both closures. They share the same function body, but share different lexical environments.

Events

What is it?

Events are occurences that happen in the browser or system which tell it that an action has or is about to commence.

Why use it?

  • Make websites and web apps more interactive
  • Provide a meaningful response for users.

How to use it? 
        
          const button = document.querySelector("#submit-button");
          button.addEventListener('click', () => {
            alert('Your form has been submitted!');
          });

Here we are simply alerting the user that their form has been successfully submitted after they have clicked the submit button. The event was fired once the user clicked the button and a meaningful response was triggered.

DOM

What is it?

The DOM is a representation of HTML and XML documents and allows programmers to interact with its structure, style, and content.

Why use it?

  • To alter aspects HTML or XML.
  • Without it, JavaScript wouldn't have any model of HTML and its entities.

How to use it? 
        
        < h1 > Hello World < / h1 >
        < script >
          const h1Tag = document.querySelector('h1');
          h1Tag.innerHTML = 'Hey there!';
        < / script >

In the code above, we are grabbing the h1 tag and changing its text to display something different. We are interacting and altering some part of the DOM by using JavaScript.

The Module Pattern

What is it?

The Module Pattern was first designed as a way to provide both private and public functions inside of classes. Since JavaScript doesn't have classes, the module pattern mimicks this notion.

Why use it?

  • Provide a standard code design to follow.
  • Protect against collisions in the global namespace.

How to use it? 
        
          var testModule = (function () {

          var counter = 0;

          return {

            incrementCounter: function () {
              return counter++;
            },

            resetCounter: function () {
              console.log( "counter value prior to reset: " + counter );
              counter = 0;
            }
          };

        })();

        // Usage:

        // Increment our counter
        testModule.incrementCounter();

        // Check the counter value and reset
        // Outputs: counter value prior to reset: 1
        testModule.resetCounter();

In this example, we defined a variable testModule and set it to an IIFE. Inside this IIFE we initialize a variable counter and return a function that increments count by one. We also set a resetCounter function which if invoked would set the variable count to zero. Calling testModule.incrementCounter() increases the counter variables by one, then we are resetting it to zero by invoking testModule.resetCounter().

This

What is it?

The this keyword refers to the current function being invoked. Its default is the Window object.

Why use it?

  • To refer to the current function being invoked.
  • To refer to the Window object

How to use it? 
        
          var thisFunction = {
              props: 10,
              func: function() {
                return this.props;
            }
          }
          thisFunction.func();

Invoking thisFunction.func() will result in an ouput of 10. That is because the inner function func is using the this keyword to refer to the outer functions property and thus returns the value of props. which is ten.
